# v0.11.0 - Invocation-native 运行时

状态：破坏性版本迁移说明

本版本将 MCO 从 findings/决策流水线改为原始回答优先的 invocation 运行时。父规格和历史版本说明保持不变；本文面向迁移到新公共接口的调用方。

## 新行为

- 使用可重复的 `--agent [alias=]provider:model` 表达明确的 Agent 身份，并支持同一 Provider 的多个模型。
- 使用 `--providers` 作为每个 Provider 选择一个配置/默认模型的 invocation-native shorthand。
- 消费原始回答正文和运行状态。任务状态只有 `complete`、`partial`、`failed`，退出码分别为 `0`、`1`、`2`。
- 使用文本模式、`--json` 或 `--stream jsonl`；stdout 只放回答或选定机器协议，诊断信息进入 stderr。
- 使用 `--chain`、`--debate`、`--synthesize` 运行由 LLM 驱动的文件化阶段。
- 使用 `--result-mode artifact|stdout|both` 控制临时或持久 Markdown 产物。
- synthesis manifest 会保留 run 与 debate 的记录（包括失败和缺失项），但绝不包含 synthesis 自身。根 `result.md` 按实际阶段分组，将 synthesis 置顶，并保留每一份原始回答和失败记录。

## 破坏性移除

以下接口已移除：

- `mco findings` 和 findings JSON schema；
- semantic normalization、去重、confidence、consensus、passive lifecycle、findings-driven memory runtime；
- Markdown-PR 和 SARIF 渲染器；
- 基于内容的 `INCONCLUSIVE` 状态；
- `--format`、`--strict-contract`、`--memory`、`--space`、`--diff`、`--staged`、`--unstaged`、`--diff-base`。

旧参数会返回迁移提示，不会被静默忽略。

## 迁移示例

旧的结构化审查调用应把范围和输出要求写进自然语言 prompt：

```bash
mco review --target-paths runtime --providers claude,codex \
  --prompt "审查 runtime/ 的正确性和安全风险，用自然语言说明推理。" \
  --result-mode both --task-id runtime-review
```

旧的多模型调用应改为显式 invocation：

```bash
mco run --prompt "比较这两个实现。" \
  --agent fast=pi:model-a \
  --agent careful=pi:model-b \
  --stream jsonl
```

旧的持久化 memory 调用应改为保存 Markdown，并通过 chain/debate/synthesis 阶段传递：

```bash
mco run --prompt "分析这个仓库。" \
  --providers claude,codex --chain --result-mode artifact --task-id repository-analysis
```

## 写入安全

MCO 不创建或管理 worktree。用户显式选择并行写入时，调用方 Agent 必须用不重叠的 `--target-paths` 划分范围，并提示文件编辑冲突风险。
